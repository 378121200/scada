Configuración de comunicación con dispositivos

La interacción con los dispositivos reales o virtuales se realiza por la aplicación llamada Comunicador que funciona como un máster o un esclavo, solicita los datos y envía comandos a los dispositivos. Todos los dispositivos están vinculados con las líneas de comunicación. Las líneas de comunicación son independientes entre sí y funcionan en paralelo.

La interfaz de usuario del Comunicador, siendo diseñada para la configuración, está integrada en la aplicación llamada Administrador. El Comunicador está configurado como parte de un proyecto.

Imagen 1 muestra un ejemplo de parámetros de la línea de comunicación principal. Un canal de comunicación define la interfaz física o un protocolo de red que se usa para intercambiar los datos con los dispositivos. Se apoyan los siguientes canales de comunicación: Serial port, TCP client, TCP server y UDP. A veces si la interacción con los dispositivos se realiza por un controlador del dispositivo, se requiere que el canal de comunicación esté indefinido (p.ej., el controlador OPC).

Si no se requiere enviar comandos a los dispositivos, se recomienda desmarcar la casilla de verificación Comandos habilitados por razones de seguridad.

Imagen 2. Secuencia de solicitud

El orden de comunicación y los parámetros de solicitud están establecidos en la página Secuencia de solicitud (refiérase a la Imagen 2).

Si la casilla de verificación Activo en la página Parámetros principales está desmarcada, la línea de comunicación correspondiente está deshabilitada y las solicitudes no se realizan. Si la casilla de verificación Activo en el cuadro de grupo Dispositivo seleccionado está desmarcada, la comunicación con aquel dispositivo está deshabilitada.

La casilla de verificación Vinculado con servidor en la página Parámetros principales permite activar o desactivar el envío de los datos de la línea de comunicación al Servidor. La casilla de verificación Vinculado con servidor en el cuadro de grupo Dispositivo seleccionado tiene el mismo objetivo, aunque se aplica solamente para el dispositivo. Si la casilla de verificación Interacción con servidor en la página Parámetros comunes de los ajustes del Comunicador está desmarcada, cualquier interacción entre el Comunicador y el Servidor está deshabilitada. Estas opciones son útiles para probar nuevos dispositivos que están conectados con el sistema.

Si los parámetros de un dispositivo Hora y Período son iguales a cero, el dispositivo se solicita de manera cíclica. Si el parámetro Hora es más de cero y el Período es cero, el dispositivo se solicita una vez al día en la hora especificada. Si el Período es más de cero, el dispositivo se solicita periódicamente a partir de la hora especificada. El campo Tiempo de espera define el tiempo de espera de respuesta del dispositivo después de la solicitud. El campo Retraso define un retraso después de cada solicitud enviada al dispositivo. La Línea de comando puede contener parámetros adicionales descritos en la documentación de un controlador del dispositivo.

Para restablecer los parámetros de solicitud del dispositivo seleccionado a los valores por defecto, haga clic en el botón Restablecer. Para abrir el formulario de propiedades del dispositivo, en caso de que se soporte por un controlador del dispositivo seleccionado, haga clic en el botón Propiedades o use un menú emergente del explorador de proyectos. Para establecer las propiedades globales para un tipo de dispositivo, elija la página Controladores, seleccione un controlador del dispositivo y haga clic en el botón Propiedades si está habilitado.

Para importar las líneas de comunicación y los dispositivos de la base de datos de configuración a los ajustes del Comunicador, haga clic derecho en el nodo Líneas de comunicación o un nodo de una línea de comunicación específica en el explorador de proyectos y seleccione un elemento Importar en el menú contextual. El formulario de importación se muestra en la Imagen 3.

La gestión de la función de sincronización de ajustes (refiérase a la Imagen 4) se realiza usando el menú contextual de la línea de comunicación. La sincronización permite actualizar los parámetros de las líneas de comunicación existentes y los dispositivos según la base de datos de configuración. Sin embargo, se puede que los parámetros ingresados a mano se pierdan.
