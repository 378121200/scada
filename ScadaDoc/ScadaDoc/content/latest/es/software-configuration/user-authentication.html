Configuración de autenticación del usuario

Rapid SCADA apoya tres métodos de autenticación del usuario:

  1. Basando en los nombres de usuarios y las contraseñas almacenadas en la base de datos de configuración.
  2. Basando en Active Directory.
  3. Método combinado.

Para realizar la autenticación, una aplicación de cliente, por ejemplo, Comunicador o Estación Web, envía una solicitud a la aplicación Servidor para que valide el nombre de usuario y la contraseña. El Servidor verifica los credenciales del usuario y devuelve el rol del usuario a la aplicación de cliente.

Los roles de usuario estándar y sus capacidades se enumeran en la tabla a continuación.

ID Nombre de rol Descripción
0 Disabled El acceso al sistema es denegado
1 Administrator Acceso completo
2 Dispatcher Visualización de toda la información, envío de comandos
3 Guest Visualización de toda la información
4 Application Interacción con la aplicación Servidor

Para restringir el acceso del usuario respecto a los objetos de la interfaz (las visiones de tablas, esquemas, etc.), cree nuevos roles de usuario en la tabla Roles en la base de datos de configuración. Luego especifique los derechos de acceso en la tabla Derechos.

Si Rapid SCADA opera en una red gestionada por Active Directory, entonces se recomienda usar los métodos de autenticación 2 y 3 debido a razones de seguridad. Los detalles de estos métodos se describen a continuación.

Para hacer que e servicio del Servidor interactúa con Active Directory, especifique la ruta del controlador de dominio y marque la casilla de verificación cercana en la página Parámetros comunes de la aplicación y habilite ModActiveDirectory.dll en la página Módulos.

El 2do método de autenticación se usa si los roles estándar son suficientes para gestionar los derechos del usuario. La ventaja de este método es que la gestión de derechos se realiza usando las herramientas de Active Directory estándar sin editar la base de datos de configuración y sin reinicar el servidio de Servidor.

Para utilizar el 2do método se requiere crear los grupos de seguridad en Active Directory. Los grupos corresponden a los roles de usuarios:

  • ScadaDisabled - rol Disabled;
  • ScadaAdmin - rol Administrator;
  • ScadaDispatcher - rol Dispatcher;
  • ScadaGuest - rol Guest;
  • ScadaApp - rol Application.

Si un usuario es un miembro de un grupo enumerado anteriormente o un miembro de un grupo que a su vez es un miembro de los grupos mencionados anteriormente, entonces los derechos correspondientes en Rapid SCADA están concedidos al usuario.

El 3er método combina las capacidades de los primeros dos métodos. La validación de los credenciales del usuario se realiza usando Active Directory y un rol de usuario se define por la tabla Usuarios de la base de datos de configuración. En este caso los nombres de usuarios y los roles de usuarios están especificados en la tabla Usuarios, aunque las contraseñas de usuarios siguen en blanco en la tabla.

Se permite el uso simultaneo de todos los métodos de autenticación mencionados anteriormente.
