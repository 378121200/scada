Conceptos básicos de configuración

Creación de proyecto

La configuración de Rapid SCADA se realiza a base de un proyecto. Un proyecto es un conjunto de archivos en varios formatos que están almacenados en el directorio de proyecto. Para crear y editar los proyectos utilice la aplicación llamada Administrador. Al iniciar el Administrador se abre la Página de inicio que contiene los botones para crear un proyecto nuevo o abrir uno existente (refiérase a la Imagen 1).

Estructura de proyecto

La configuración de Rapid SCADA se visualiza en el explorador de proyectos que se ubica en la parte izquierda de la ventana principal del Administrador. El proyecto consta en las siguientes partes principales (Imagen 3):

  • La base de datos de configuración es una descripción estructurada de todo el sistema automatizado. La base de datos de configuración define el sistema en general, mientras que los ajustes detallados se refieren a las aplicaciones de Rapid SCADA correspondientes.
  • Interfaz. Esta sección de configuración contiene los archivos de visiones y los archivos de configuración de informes. Los ejemplos de visiones: los esquemas, las tablas, los mapas y los paneles de instrumentos.
  • Los ajustes de la aplicación Servidor.
  • Los ajustes de la aplicación Comunicador.
  • Los ajustes de la aplicación web Estación Web. Es mucho más cómodo editar los ajustes de la Estación Web a través de la interfaz web y luego descargarlos en el proyecto.


Imagen 3. Estructura de proyecto

Una instancia es un ordenador donde está desplegado Rapid SCADA. Un solo proyecto puede incluir múltiples instancias de Rapid SCADA que intercambian los datos. La aplicación Administrador puede conectarse con los servidores remotos para descargar y subir la configuración, así es posible configurar Rapid SCADA usando una estación de trabajo.

Secuencia de configuraciones general

Al iniciar el trabajo con Rapid SCADA se recomienda seguir la secuencia de configuraciones general descrita a continuación. Al obtener más experiencia y mejor comprensión de las dependencias entre las aplicaciones, se podrá cambiar la secuencia para aumentar la eficiencia.

  1. Cree un proyecto nuevo o abra uno existente.
  2. Añada los objetos, las líneas comunicaciones y los dispositivos en la base de datos de configuración. Para crear las líneas de comunicaciones y los dispositivos se recomienda usar los asistentes que se abren usando los botones y ubicados en la barra de herramientas del Administrador.
  3. Verifique que las líneas de comunicaciones y los dispositivos añadidos están presentes en los ajustes del Comunicador. Si es necesario, impórtelos seleccionando el elemento del menú Importar del menú contextual del nodo Líneas de comunicaciones del explorador de proyectos.
  4. Configure la comunicación con los dispositivos en el Comunicador.
  5. Cree los canales de entrada y salida en la base de datos de configuración. Se recomienda usar el asistente que se abre usando el botón .
  6. Cree las visiones y especifíquelos en la tabla Interfaz de la base de datos de configuración.
  7. Suba el proyecto al servidor usando el botón .
